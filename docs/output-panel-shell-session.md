# 输出面板「终端化」方案（续跑同一 Tab）

> 目标：在现有输出面板上，用最小代价模拟「地址栏同步的终端 REPL」，而非完整 PTY 终端。

## 一、目标与边界

### 要达成的体验

| 需求 | 含义 |
|------|------|
| 同一 Tab 续跑 | 底部命令框回车 → **当前选中 Job** 追加输出，不 `createJob` |
| 追加输出 | 保留历史，命令之间加分隔 |
| 终止 | 运行中可打断当前 Job 的子进程 |
| 命令框快捷键 | Cmd+A / C / V 等标准编辑 |
| 目录同步 | **每次执行** 的工作目录 = **当前窗口地址栏路径** |
| 环境 & 变量 | 尽量接近本机 zsh；支持 `%d` 等 Snippet 占位符 |

### 明确不做（本阶段）

- **不是** PTY 长驻 shell（`vim`、`htop`、`read -p` 仍不适合）
- **不是** 命令间 shell 内 `cd` 持久保留（每次新 `Process`，靠地址栏注入 `cd`）
- **不是** 完整 ANSI 终端仿真（可二期加颜色）

#### 哪些命令适合 / 不适合在命令框里跑？

本命令框本质是：**每次回车都启动一个新的 `zsh -ilc "cd 'cwd' && 命令"` 进程，一次性跑完就退出**，所以支持度可以按下面来理解：

| 类型 | 示例 | 支持情况 |
|------|------|----------|
| 一次性执行就结束的命令 | `ls`、`pwd`、`cat file`、`echo hi`、`grep ...`、`find ...`、`git status`、`python3 -c '...'` | ✅ 完全支持 |
| 文件/目录操作 | `mkdir`、`rm`、`cp`、`mv`、`touch` | ✅ 完全支持 |
| 只关心退出码的命令（无输出也可） | `cd some-dir`、`true`、`test -f file` | ✅ 完全支持，结束后有 `✓` / `✗` 标记 |
| 清屏 | `clear` | ✅ 已做成内置清屏，不启动 shell |
| 依赖 `PATH` 的普通 CLI 工具 | `rg`、`fd`、`xz`、`tar` 等 | ✅ 只要本机 PATH 找得到，就和终端一致 |
| 修改环境变量 | `export FOO=bar`、`alias ll='ls -la'` | ⚠️ 只对**当前这次进程**有效，下一条命令会重新启动新的 `zsh`，效果不会保留 |
| 持续运行 / 守护 / 监听 | `python -m http.server`、`node server.js`、`webpack-dev-server` 等 | ⚠️ 技术上可以启动，但会**长时间占用该 Tab**，不推荐；若要跑请注意用 Ctrl+C 终止 |
| 需要交互输入的命令 | `read -p ...`、交互式安装脚本、需要在同一终端内输入密码/菜单选择的程序 | ❌ 非 PTY，交互体验非常差或直接挂起，不推荐 |
| 全屏 / 富 TUI 程序 | `vim`、`nvim`、`nano`、`less`、`more`、`top`、`htop`、`fzf` 等 | ❌ 非 PTY，界面无法正常渲染，不支持 |
| 复杂 ANSI / 颜色控制 | 彩色日志、简单 ANSI 颜色 | ⚠️ 纯文本仍能显示，**颜色控制码不会被完整仿真** |

> 心智模型：把它当成「附带当前地址栏 cwd 的 batch 模式 `zsh`」，适合**跑脚本 / 工具 / 一次性命令**，不适合「在这里开一个真正的终端会话」。

本质：**「地址栏同步的、可重复提交的 zsh 批处理 REPL」**。

## 二、现状与差距

- `rerunEditedCommand` 固定 `createJob` → 需改为原地调度
- `ShellRunner` 使用 `zsh -lc`，未显式继承环境
- `OutputPanelView` 未接收 `path` / `selection`，无法展开 `%d` 等
- 底部框为 SwiftUI `TextField`，快捷键弱

---

## 三、架构

```
ContentView (path, selection)
    → OutputExecutionContext
    → OutputPanelView
    → 底部 NSTextField 回车
    → JobStore.executeInPlace(jobID, raw, context)
    → SnippetExpander.expand
    → ShellRunner (zsh -ilc, cd cwd, append output)
```

### Shell 执行

| 项 | 方案 |
|----|------|
| 解释器 | `/bin/zsh` |
| 参数 | `["-ilc", "cd 'cwd' && expanded"]` |
| 环境 | `ProcessInfo.processInfo.environment` |
| cwd | `currentDirectoryURL` + 命令内 `cd` |

### 键盘

| 焦点 | 快捷键 | 行为 |
|------|--------|------|
| 命令框 | Cmd+A/C/V | 系统默认编辑 |
| 运行中 | Ctrl+C | `cancel(jobID)` |
| 输出区运行中无选区 | Cmd+C 可选映射终止 | 与复制分工 |

## 五、命令框配色（琥珀终端风）

与输出区（冷灰底 + 青色提示符）形成对比，命令输入条使用**暖琥珀色**：

| 元素 | 色值 | 说明 |
|------|------|------|
| 背景 | `#292417` | 深琥珀褐，比输出区略暖 |
| 文字 | `#FFE06B` | 高亮琥珀黄，等宽 medium |
| 边框 | `#E6AE38` | 默认金边 |
| 聚焦边框 | `#FFD152` | 更亮的聚焦反馈 |

查找框仍用系统浅灰胶囊，避免与命令框抢视觉焦点。

## 六、Tab 补全（参照终端）

实现文件：`OutputCommandCompleter.swift`

### 行为（对齐 bash/zsh 常见体验）

| 场景 | Tab 行为 |
|------|----------|
| 唯一匹配 | 直接补全 |
| 多匹配有公共前缀 | 先扩展到最长公共前缀 |
| 多匹配无公共前缀 | 列出候选 + 填入第一项；再按 Tab 轮换 |
| 首词 | 补全 `PATH` 中的命令名 |
| 含 `/` 或 `~` | 路径补全（目录加 `/`） |
| 后续词 | 当前目录文件名补全 |

### 实现策略

1. **优先 zsh**：`zsh -ilc` 跑补全脚本（与用户执行环境一致），超时 350ms 回退
2. **Swift 回退**：扫描 `PATH` + `FileManager` 列目录
3. **会话状态**：`OutputCommandCompletionSession` 记录候选与轮换索引
4. **候选展示**：多匹配时在命令框上方一行显示（类似终端列出选项）

### 明确不做

- 完整 `compinit` / 子命令选项补全（如 `git checkout` 分支名）
- 异步补全 UI（二期可考虑后台 zsh + 弹出菜单）

## 七、性能

| 维度 | 影响 |
|------|------|
| 空闲 | 几乎无 |
| 单次执行 | `-ilc` 约 +50–200ms |
| 内存 | 少 Tab；单 Job 字符串变长需上限 |
| 主窗口 | 无影响 |

## 八、验收清单

1. 底部回车：当前 Tab 追加，不新开 Tab
2. 地址栏切换后 `pwd` 正确
3. `%d` / `%p` 等展开正确
4. 命令框 Cmd+A/C/V 正常
5. Snippet 首次执行仍可新 Tab
6. 运行中 Ctrl+C 终止当前 Tab 命令
7. stderr 红色、stdout 浅灰、提示符高亮；输出区深色背景
8. 单 Tab 输出超过约 1.5MB 时截断并提示
9. 命令框琥珀配色醒目；↑/↓ 历史；Tab 补全命令与路径
10. 命令框 `cd` 同步主窗口地址栏与文件列表
11. 无 stdout/stderr 时仍显示 `✓` / `✗` / `⊘` 状态标识（着色区分；退出码见底部栏）
12. stderr 按命令发生顺序内联显示，不堆在后续命令下方
13. 输入 `clear` 清空当前 Tab 输出（等同终端清屏，无清屏按钮）
14. 测试通过
