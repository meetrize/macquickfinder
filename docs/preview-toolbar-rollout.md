# 预览标题栏按钮与分期实施方案

> 目标：在不引入重依赖、不明显增加内存与响应负担的前提下，逐步扩展文件预览能力，并为不同类型提供高频标题栏操作按钮。

## 1. 设计原则

- 以系统能力为主：`PDFKit`、`AVKit`、`QuickLook`、`Foundation`。
- 不引入重依赖：首期不集成 LibreOffice、FFmpeg、Unrar 等库。
- 按“可取消异步加载”实现：切换文件立即取消旧任务。
- 标题栏按钮统一模型：`button / toggle / menu / stepper`。
- 按文件类型分层降级：可原生渲染则原生，不能原生则 QuickLook，仍不可用则外部打开。

## 2. 文件类型与按钮矩阵（最小实现）

### 2.1 首期可做（低风险）

- 图片（`jpg/png/heic/tiff/webp/gif`）
  - 预览：原生图片预览
  - 标题栏按钮：放大、缩小、适配窗口、100% 还原、旋转 90 度（仅视图）
  - 性能要点：按视图尺寸解码，避免全尺寸常驻

- PDF（`pdf`）
  - 预览：`PDFView`
  - 标题栏按钮：上一页、下一页、放大、缩小、适配宽度/整页
  - 性能要点：不做全量缩略图预渲染，按页按需显示

### 2.2 次阶段可做（中低风险）

- 文本/代码（`txt/md/csv/json/xml/yaml/log/sql` 及常见代码扩展）
  - 预览：文本预览
  - 标题栏按钮：自动换行、高亮开关、复制全文、跳转顶部/底部
  - 性能要点：保留截断策略，超大文件自动关闭高亮

- 音视频（`mp4/mov/mp3/wav`）
  - 预览：`AVPlayerView`
  - 标题栏按钮：播放/暂停、静音（倍速可选）
  - 性能要点：不做转码、波形分析、自动抽帧

### 2.3 后续可做（有条件）

- Office（`docx/xlsx/pptx`）
  - 预览：QuickLook
  - 标题栏按钮：用默认应用打开、重新加载
  - 说明：不做自研 OOXML 渲染

- 压缩包（`zip/tar/gz/tgz`）
  - 预览：至少展示内部目录
  - 标题栏按钮：刷新目录、展开/折叠、复制清单
  - 说明：首版不做内部文件解压预览

- `rar/7z`
  - 预览：系统工具能力探测后列目录
  - 标题栏按钮：列目录（工具可用时）
  - 说明：无工具时清晰提示，不内置第三方解析库

### 2.4 首期不做（高成本高风险）

- `dwg/dcm/fig/sketch` 自研渲染
- 压缩包内全文索引、自动递归预览
- 专业编辑型功能（PDF 注释编辑、图像写回）

## 3. 分期实施方案

## P1：图片 + PDF 按钮与性能基线

### 范围

- 图片标题栏补齐（适配/还原可后续补）
- PDF 标题栏新增（上一页/下一页）
- 图片、PDF 读取迁移为后台任务

### 验收

- 切换文件时按钮状态不串
- 图片/PDF 打开时主线程无明显阻塞
- PDF 上下页按钮随页码边界自动禁用

## P2：文本/代码 + 音视频

### 范围

- 文本类按钮（换行、高亮、复制、跳转）
- 音视频按钮（播放/暂停、静音）

### 验收

- 大文本不会卡 UI，且有截断提示
- 播放状态与标题栏状态同步

## P3：Office + 压缩包目录

### 范围

- Office QuickLook 预览 + 外部打开
- 压缩包目录展示（zip/tar/gz/tgz）
- rar/7z 能力探测策略

### 验收

- 无第三方依赖也可稳定运行
- 无系统工具时可读提示，不崩溃

## P4：行业增强

### 范围

- `ipynb` 结构化只读展示
- `psd` QuickLook 与元信息
- `dwg/dcm/fig/sketch` 继续外部打开兜底

### 验收

- 无法渲染格式均可降级处理且可外部打开

## 4. 性能红线（强制）

- 主线程禁止同步读文件、解码、归档扫描、外部命令执行。
- 所有预览加载必须可取消（基于 `task(id:)`）。
- 大对象在文件切换后释放（图片、PDF、播放器实例）。
- 外部命令必须限时并支持取消。
- 缓存须有上限，不允许无限增长。

## 5. 当前代码落地点（P1）

- `Sources/Explorer/AppModule.swift`
  - `FilePreviewView`：标题栏按钮渲染
  - `FileContentView`：图片/PDF 异步加载
  - `PDFPreview`：对外暴露页码变化能力，供标题栏控制
